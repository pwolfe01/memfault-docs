This page documents the various FQDNs that the Memfault app and CLI use, and the
kind of content used by each endpoint.

### Memfault FQDNs

| FQDNs                                 | Usage                                                        |
| ------------------------------------- | ------------------------------------------------------------ |
| api.memfault.com (Web)                | <a href="https://api-docs.memfault.com/">Memfault API</a>    |
| app.memfault.com (Web)                | <a href="https://app.memfault.com/">Memfault app</a>         |
| chunks.memfault.com (MCU)             | <a href="https://mflt.io/data-to-cloud">Uploading Chunks</a> |
| device.memfault.com                   | Uploading events (from a device)                             |
| files.memfault.com                    | Uploading files (from a device)                              |
| ingress.memfault.com (CLI, Cloud SDK) | Uploading coredumps, events manually                         |
| memfault-prod-east1.s3.amazonaws.com  | Serving file assets                                          |
| ota-cdn.memfault.com                  | Serving OTA assets                                           |
